Configuring FTP accounts

Automatic folder creation

ShareX can automatically create sub-folders in your FTP Server based on the SubFolderPath settings. FTP Path of a file is shown below:

URL = ftp:// + Host + : + Port + SubFolderPath + FileName

URL creation based on FTP Account settings

The usual URL creation procedure is as follows: 

URL = HttpHomePath + SubFolderPath + FileName

This default behavior of URL creation can be customized.

If HttpHomePath starts with the @ char then,

URL = HttpHomePath + FileName

If HttpHomePath is empty then,

URL = Host + SubFolderPath + FileName

Note: If URL does not start with http:// then it will be added automatically. The / char will be handled automatically; you don't need to add / char to the beginning or end of paths.

If Host starts with ftp. then it will be automatically removed while making URL (when using empty HttpHomePath or when using % in HttpHomePath).

Standard usage

Using IP address

Host: 80.1.3.37
SubFolderPath: ShareX
HttpHomePath: getsharex.com/jaex
PreviewFtpPath: ftp://80.1.3.37:21/ShareX/screenshot.jpg
PreviewHttpPath: http://getsharex.com/jaex/ShareX/screenshot.jpg

Using domain

Host: ftp.getsharex.com
SubFolderPath:
HttpHomePath:
PreviewFtpPath: ftp://ftp.getsharex.com:21/screenshot.jpg
PreviewHttpPath: http://getsharex.com/screenshot.jpg

Using subdomain

Host: jaex.getsharex.com
SubFolderPath: ShareX/Screenshots
HttpHomePath: www.getsharex.com/jaex
PreviewFtpPath: ftp://jaex.getsharex.com:21/ShareX/Screenshots/screenshot.jpg
PreviewHttpPath: http://www.getsharex.com/jaex/ShareX/Screenshots/screenshot.jpg

Using empty HttpHomePath

Using domain

Host: ftp.getsharex.com
SubFolderPath: ShareX
HttpHomePath:
PreviewFtpPath: ftp://ftp.getsharex.com:21/ShareX/screenshot.jpg
PreviewHttpPath: http://getsharex.com/ShareX/screenshot.jpg

Using subdomain

Host: jaex.getsharex.com
SubFolderPath: Screenshots/ShareX
HttpHomePath:
PreviewFtpPath: ftp://jaex.getsharex.com:21/Screenshots/ShareX/screenshot.jpg
PreviewHttpPath: http://jaex.getsharex.com/Screenshots/ShareX/screenshot.jpg

Using %host in HttpHomePath

%host char will be automatically replaced with Host.

Using domain

Host: getsharex.com
SubFolderPath: ShareX
HttpHomePath: %host/jaex
PreviewFtpPath: ftp://ftp.getsharex.com:21/jaex/ShareX/screenshot.jpg
PreviewHttpPath: http://getsharex.com/jaex/ShareX/screenshot.jpg

Using subdomain

Host: jaex.getsharex.com
SubFolderPath: ShareX
HttpHomePath: %host
PreviewFtpPath: ftp://jaex.getsharex.com:21/ShareX/screenshot.jpg
PreviewHttpPath: http://jaex.getsharex.com/ShareX/screenshot.jpg

Using @ in HttpHomePath

SubFolderPath won't be automatically appended to HttpHomePath if HttpHomePath does start with @

Using subdomain

Host: ftp.getsharex.com
SubFolderPath: ShareX
HttpHomePath: @jaex.getsharex.com
PreviewFtpPath: ftp://ftp.getsharex.com:21/ShareX/screenshot.jpg
PreviewHttpPath: http://jaex.getsharex.com/screenshot.jpg
