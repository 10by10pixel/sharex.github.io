Supported command line arguments

-multi or -m allows ShareX to load multiple instances. By default not possible to open more than one ShareX instance.

-silent or -s loads ShareX to tray without showing the main window.

-portable or -p loads ShareX in portable mode. In portable mode, the application will create a folder (if it doesn't exist already) in its startup path to save configuration files and images.

-clipboardupload performs clipboard upload as soon as ShareX opens.

"filepath" uploads file.
